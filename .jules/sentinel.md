## 2025-03-01 - Enhance AndroidManifest.xml Security Defaults
**Vulnerability:** The application had `android:allowBackup="true"` and lacked explicit `android:usesCleartextTraffic="false"` in its manifest. While not immediate exploits, they act as attack vectors. Permitting backups on a device managing API keys and local LLM chats risks unintentional cloud leakage or adb extraction. Implicit cleartext traffic allowance opens up potential MITM vectors.
**Learning:** Default generated Android manifests often prioritize developer convenience (easy backups) over strict security by default. In an AI context where chat history and credentials exist locally, these must be explicitly locked down.
**Prevention:** Always default to `android:allowBackup="false"` and `android:usesCleartextTraffic="false"` in the primary manifest. If backups are genuinely needed, create strict, granular `data_extraction_rules` targeting only non-sensitive preferences, excluding databases and EncryptedSharedPreferences.
