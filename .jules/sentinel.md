## 2026-04-12 - [Path Traversal in Model File Scanner]
**Vulnerability:** Path traversal vulnerability in `ModelFileScanner.kt` due to lack of filename sanitization when dynamically resolving paths using `File(modelsDir, artifact.localFileName)` during operations like file validation and deletion.
**Learning:** In Android apps where file structures heavily interact with user or network configured model filenames, raw string interpolation in `File(parent, child)` fails to prevent parent directory (`../`) traversals. Private class-level extension functions mapping directly over Coroutine/Dispatcher boundaries (like inside `withContext(Dispatchers.IO)`) can trigger `Unresolved reference` issues.
**Prevention:** Always extract and use only the file's basename component via `File(filename).name` and strictly validate the canonical path against the expected parent's canonical path before file access. Expose these as explicit private helper functions (e.g. `isSafeChildOf`) rather than class extensions to prevent Kotlin compiler resolution issues in suspending coroutine blocks.
